# SHOGI-TOUR A-5.0 認知整理メモ v0.1

## 0. このメモの位置付け

- 正式仕様書ではない
- 髙橋実体験ベースで作成する初版（v0.1）
- 不足部分は岩崎さん・磯崎さんへの確認で補完（v0.2）
- A-5.1 以降の戦略判断は ChatGPT 司令塔復帰後に実施
- 外販可能性は記録するが本フェーズでは判断しない
- 司令塔冗長化ルール v1.0 §7 遵守（代行禁止範囲に該当する判断は含まない）

## 1. A-5.0 の目的

- 大会運営の現場認知を棚卸しする
- A-5.1 以降の状態モデル設計の土台を作る
- 髙橋依存からの段階的脱却
- 「仕様確定」ではなく「現場認知の棚卸し」と位置付ける

## 2. 成果物4種

### 2.1 役員ジャーニーマップ

- フォーマット: 時刻 × 役員動作 × 思考 × 焦り度 × アプリ介入余地
- 作成: 髙橋実体験ベース（50年将棋＋幹事経験）
- 期待行数: 30〜50 行
- 完了目安: v0.1 を 1 週間以内

### 2.2 状態遷移図

- 3 レイヤー: 大会全体／参加者個別／対局カード
- 形式: テキスト or 簡易図
- 完了目安: 3〜5 日

### 2.3 失敗パターン集

- フォーマット: 事例 N／状況／根本原因／対処／再発防止／アプリ解決可否
- 期待件数: 5〜15 件（まずは思い出せる範囲）
- 完了目安: 1 週間以内（他作業並行）

### 2.4 役員言語辞書

- フォーマット: 用語／意味／仕様書言語／備考
- 期待件数: 30〜50 語
- 完了目安: 1 週間以内（他作業並行）

## 3. 進め方

### Week 1: 髙橋実体験ベース v0.1 作成

- ジャーニーマップ初版
- 状態遷移図初版

- v0.1 で不足が判明した部分のみ

- A-5.0 完了報告 docs を `docs/specs/` に作成

### 期間固定しない

- 1 週間で十分なら即 A-5.1 へ移行
- 不足があれば追加整理

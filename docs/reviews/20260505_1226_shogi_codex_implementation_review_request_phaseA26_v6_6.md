## 1. 背景

### 1.1 Phase A-2.6 の発端

A-2.5 main マージ後、髙橋さんが iPhone Safari で実機検証中に **3つの UX バグ** を発見：

1. **F1**: マイグレモーダル「統合を実行」ボタンが iPhone で反応しない
   - 原因: iOS Safari の textarea blur 競合（テキストエリアにフォーカスがある状態で他要素タップ → 1回目が blur に消費）
   - **動作確認済**: テキストエリア外を一度タップしてキーボードを消してから「統合を実行」をタップ → 動く（仮説 B 確定）
   - **Mac 検証済**: Mac Safari/Chrome では完全に正常動作

2. **F2**: マイグレモーダルの textarea で自動ズームが発生
   - 原因: `font-size:12px` が iOS Safari の自動ズーム閾値（16px）未満
   - 症状: テキストエリアタップ時に画面拡大、ボタンが画面外に押し出される

3. **F3**: 読み込みモーダルの textarea でも同じ自動ズーム問題（推定）
   - 原因: `font-size:13px` も 16px 未満
   - コードベースで F2 と同じパターン

### 1.2 修正内容（PR #5、+3/-2 行のみ）

| ID | 修正 | 変更行数 |
|---|---|---|
| F1 | `bindMigrationModalEvents` の runBtn ハンドラ冒頭に `if(textarea) textarea.blur();` 追加 | +1 |
| F2 | `mig-paste-area` の `font-size: 12px → 16px` | +1/-1 |
| F3 | `load-paste-area` の `font-size: 13px → 16px` | +1/-1 |

データモデル変更なし、構造変更なし、新規テスト追加なし。

### 1.3 ChatGPT メタレビュー省略の判断

| 理由 | 内容 |
|---|---|
| 対症療法 | iOS Safari の既知挙動への対応で議論の余地が少ない |
| 修正規模が極小 | 数行のみ |

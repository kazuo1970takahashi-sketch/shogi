## 2. Minor 推奨（5件、実装プロンプトで反映）

### M1. 「キャンセル」と「空欄追加」のUI文言明確化

ダイアログの説明強化：
- 「ふりがなを入れると、次回から50音タブ・ふりがな検索で探しやすくなります。空欄のままでも追加できます。」
- 「キャンセルすると、選択した参加者は追加されません。」

### M2. normalizeYomi 境界ケース

A-2 では半角カナ非対応を明記。テスト追加：
- ヴ / ヴァン系
- ヵ / ヶ
- 半角カナ（ｻﾄｳ）→ A-2 では非対応 = other 扱い
- 先頭長音符・記号・数字 → other

### M3. 「他」タブの補足文

UI に「他: ふりがな未登録・その他」を添える。

### M4. 全員空欄追加時の通知

showMsg で軽い通知：
- 「ふりがな未登録のまま追加しました。未登録者は『他』タブに表示されます。」

### M5. 既存テスト数の表記統一

実装依頼で「既存全161件、うち支部マスタ111件を緑維持」と統一。

## 3. 主要論点（v6 §1.3）への評価

### 1. ふりがな入力方式 Y（vs X / Z）

**判定**: 採用妥当

- 方式 Y が最適
- 方式 X は F7 マスタ編集画面が欲しくなり A-2 が膨らむ
- 方式 Z は新規参加者登録フローまで変えて広い

### 2. ダイアログ形式 B（vs A / C）

**判定**: 採用妥当

- B が最適
- A は連続ポップアップでスマホ運用ストレス
- C は yomi 入力が後回しになり F5b/F6 を入れる意味が薄れる

### 3. yomi 空の「他」集約（vs 非表示）

**判定**: 採用妥当

- 「他」集約が正しい
- 非表示は「壊れた」誤解の危険がある
- A-1 既存資産保護の観点で「他」集約が安全

## 4. ChatGPT 提示の追加推奨テスト（9項目）

実装プロンプトに反映：

1. A-1 既存マスタ互換（yomi 空文字での全機能動作）
2. yomi 欠落互換（yomi プロパティ自体が undefined）
3. normalizeYomi 境界（ヴ/ヵ/ヶ/半角カナ/先頭長音符）
4. F4 ダイアログ部分入力（3名中2名入力、1名空欄）
5. F4 キャンセル（yomi も参加者リストも変更されない）
6. F6 件数保存則（全タブ件数 = 各行 + 他 の合計）※検索なしの場合
7. F6 AND 条件（タブ + 検索で 0件 / 1件）
8. crypto 通知（同一セッション過剰繰返しなし）
9. マイグレ警告（フラグあり/なし両方）

## 5. ChatGPT 提示の実装完了条件（25項目）

A-2 実装完了条件：

1. schema_version は 1 のまま
2. 既存 A-1 マスタをロードしても壊れない
3. yomi 空・yomi 欠落・null相当でも例外が出ない
4. ワンクリック呼び出し時、yomi 空の人だけダイアログに出る
5. yomi 入力済みの人はダイアログに出ない
6. ダイアログで入力した yomi は normalizeYomi 後にマスタへ保存
7. 空欄のまま追加した人は yomi 空のまま参加者リストに追加
8. キャンセル時は yomi も参加者リストも変更されない
9. 漢字検索は既存通り動く
10. ふりがな検索は yomi 部分一致で動く
11. カタカナ検索は normalizeYomi によりひらがな yomi と一致
12. ローマ字検索は A-2 では実装しない
13. 50音タブは全/あ/か/さ/た/な/は/ま/や/ら/わ/他
14. 全タブは既存同等の全件表示
15. yomi 空の人は「他」タブに表示
16. タブ + 検索は AND 条件
17. 「他」タブでは yomi 空が先頭に来る
18. 破損マスタ状態でマイグレーションウィザードを開くと警告が出る
19. crypto 非対応時はユーザーに通知
20. crypto 非対応通知は同一セッションで過剰に繰り返さない
21. A-1 既存テスト161件、特に支部マスタ111アサーションをすべて緑維持
22. 新規テストを追加し、normalizeYomi / getYomiInitialRow / F5b / F6 / F4 / Codex Minor #1/#2 を検証
23. 意図しない関数変更がないことを確認
24. 1 Stage = 1 commit を基本
25. 実装完了後、PR本文にテスト結果と変更範囲を明記

2. F7 マスタ編集画面を作らない
3. F8 エクスポート/インポートを作らない
4. ローマ字検索を実装しない

7. 既存フォーム全体を大きく再設計しない
8. 既存CSSを大きく組み替えない
9. build/bind/coordinator パターンを崩さない
10. A-1 の既存テストを変更して通すような対応をしない
11. shogi_v4.html 以外の実装ファイルを増やさない（テストファイル追加は可）
12. Codex Minor #1/#2 が想定以上に膨らむ場合は、実装を止めて別フェーズ判断に回す

## 7. 評価された設計選択

ChatGPT が特に評価した点：

- yomi を schema_version 変更なしで活用する点
- 「全」デフォルトで既存動作を守る点
- yomi 空を非表示にせず「他」に逃がす点
- ふりがな入力を強制しない点
- F7 / F8 / ローマ字検索 / 自動推定を A-3 送りにしている点

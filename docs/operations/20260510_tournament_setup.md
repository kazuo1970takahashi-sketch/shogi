## 1. 当日朝のバックアップ手順

1. 本番 URL を iPhone で開く: https://kazuo1970takahashi-sketch.github.io/shogi/shogi_v4.html
2. 「マスタ」タブを開く
3. **📤 マスタをエクスポート** ボタンをタップ
4. ダウンロード完了 alert で **ファイル名** を確認(`shogi_branch_master_2026-05-10.json` 等)
5. iPhone のダウンロードフォルダを開いて、ファイルが保存されたことを目視確認
6. ファイルサイズと内容(任意のメモアプリで開いて先頭の `{"schema_version":1` を確認)
7. **画面に表示されている「登録: N 名」のサマリー件数** を控える(リセット直前確認用)

> 失敗時: alert に「マスタの書き出しに失敗しました」と出たら、再度ボタンタップ。3 回失敗時は **DevTools 直叩き Plan B(Phase 2 仕様書 §7)** に切替。

## 2. リセット手順

1. 「マスタ」タブ → **📛 マスタをリセット** ボタン(赤枠)をタップ
2. リセットモーダル表示
3. 表示内容を確認:
   - 「現在のマスタ件数: 生存 N 名 / 削除済み M 名」が **§1 で控えた件数と一致** することを確認(不一致なら異常 → キャンセル + バックアップ再取得)
4. **「📤 マスタをエクスポートでバックアップ JSON をダウンロード済みです(必須)」** チェックボックスを ON
5. **「リセット」と入力してください** のテキスト欄に `リセット` と完全一致で入力
6. 実行ボタンが赤色に変わる(両方の条件を満たすまで disabled)
7. **実行** タップ
8. モーダルが閉じる + 画面トーストに「マスタをリセットしました(全 member 消去)」表示
9. 「マスタ」タブの一覧が空("登録された参加者はいません" 表示)になる

> キャンセル: モーダル右の「キャンセル」 → モーダルが閉じる、マスタ不変

## 3. JSON 取り込み手順

1. 「マスタ」タブ → **📥 22 名取込(5/10 大会用)** ボタン(青枠)をタップ
2. Phase 2 import モーダル表示
3. 「JSON ファイル選択」ボタン → ファイル選択 → `20260412_participants.json` を選択
4. 自動で検証実行:
   - **検証 OK** ✅ : プレビューエリアに以下表示
     - 取り込み件数: **22 名**(A 級 **18** / B 級 **4**)
     - サンプル 3 件:
       - A 級: 片山凱翔(A、city=御殿場市)
       - B 級: 高橋一雄(B、city=沼津市)等
       - city 長め: city が最も長い 1 件
   - **検証 NG**: 赤色エラーメッセージ表示(`配列長が 22 名ではありません` 等)。`実行` ボタン disabled のまま → ファイル再選択
5. **実行** タップ
6. モーダルが閉じる + トーストに「22 名(A 18 / B 4)を取り込みました」表示
7. マスタ一覧が更新され、22 名表示

> 検証失敗時の挙動:**全件ロールバック**(マスタは変更されない)。エラーメッセージで該当行を確認 → ファイル修正 → 再実行

> 実行後の異常: 件数が 22 と異なる、または別件数表示 → §6 リカバリへ

## 6. 異常時のリカバリ

### 6.1 import 失敗(検証 NG)
- マスタは変更されていない(全件ロールバック)
- エラーメッセージの行番号を確認 → ファイルを修正 → 再実行

### 6.2 リセット後 import 失敗
- マスタは空のまま
- 解決 1: §1 で取得したバックアップ JSON を **「📂 マスタをインポート」** で復元
  - モード: **「このデータで上書き(既存マスタを破棄)」**(overwrite)
  - ファイル選択 → 確認 dialog で OK → 復元完了
- 解決 2(import 自体に問題がある場合): バックアップ復元後、シンボリック原因を Claude.ai/ChatGPT に報告 + 別日 reschedule

### 6.3 完全異常(画面が表示されない / localStorage 破損)
1. 別端末/別ブラウザを開く(同じ本番 URL)
2. バックアップ JSON を **「📂 マスタをインポート」** で復元
3. 復元成功確認後、本来の端末で再 import or バックアップ JSON を新端末でそのまま運用

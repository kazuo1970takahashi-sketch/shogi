## 1. 概要

### 1.1 Phase A-2 の位置づけ

Phase A-1 で支部マスタの基本機能（漢字検索ベース）が完成・main マージ済み。
Phase A-2 では、運用試験を待たずに **ふりがな対応の3点セット** を先行実装する。

理由：
- 大会幹事の運用上、「さとうさん」のように頭で探すのが自然な動線
- 漢字検索のみだと「画数の多い人を素早く探せない」課題が残る
- F4-F5b-F6 は密結合（データ層 → 検索 → タブUI）なので一括実装が効率的

### 1.2 含む機能（5項目）

| **F4** | ふりがな入力（マスタの参加者にふりがな列を持つ）| 小 | A-1 v5 §2 で予約済み |
| **F5b** | ふりがな検索（既存 F5a の拡張）| 小 | A-1 v5 §2 |

| **Codex Minor #1** | 破損マスタ状態でのマイグレ時の確認表示 | 極小 | Codex 再レビュー §3 観点 A |
| **Codex Minor #2** | crypto 非対応ブラウザでのユーザー通知強化 | 極小 | Codex 再レビュー §3 観点 D |

- F7 マスタ管理画面のフル機能（編集・削除/tombstone・タブ内検索）

## 3. 機能仕様

### 3.1 F4: ふりがな入力（方式 Y）

**入力タイミング**：ワンクリック呼び出しの「選択した N 名を追加」ボタンを押した時点。

**動作フロー**：

```
ユーザーが過去参加者パネルでチェック → 「追加」ボタン押下
  ↓
追加対象 N 名のうち yomi 空の人だけ抽出
  ↓
yomi 空が0名 → 即座に追加完了（既存動作）
yomi 空が1名以上 → ふりがな一括入力ダイアログ表示
  ↓
ダイアログ：参加者ごとに入力フィールド（空欄可）
  [追加する] [キャンセル]
  ↓
[追加する] → 入力された yomi を該当 member に保存（マスタ更新）
            空欄の人は yomi 空のまま
            参加者リストに追加
[キャンセル] → 何もしない（既存動作維持）
```

**ダイアログ仕様（パターン B：まとめて一括入力）**：

```
┌─ ふりがなを入力してください ──────────┐
│ （空欄のままでも追加できます）           │
│                                        │
│ 佐藤太郎  → [さとうたろう       ]       │
│ 鈴木一郎  → [                  ]       │
│ 田中健一  → [たなかけんいち    ]       │
│                                        │
│ [追加する]  [キャンセル]                │
└───────────────────────────────────────┘
```

**実装上の注意**：
- yomi が既に登録されている member は **ダイアログに含めない**（再入力不要）
- 空欄のまま「追加する」を押した人は yomi 空のまま追加（強制しない）
- yomi 入力欄は `inputmode="kana"` でスマホでかな入力モードを推奨
- 入力された yomi は `normalizeYomi(yomi)` で正規化（後述 3.4）

### 3.2 F5b: ふりがな検索（既存 F5a の拡張）

既存の検索ボックスを拡張：

**動作仕様**：
- 検索ボックスへの入力は、漢字一致 OR ふりがな一致のどちらかでヒット
- 例：
  - 「さとう」→ name=「佐藤太郎」（漢字不一致）+ yomi=「さとうたろう」（ふりがな一致）→ ヒット
  - 「佐藤」→ name=「佐藤太郎」（漢字一致）→ ヒット
  - 「sa」→ どちらでも一致しない → ヒットせず（A-2 ではローマ字検索しない）
- 検索文字列も `normalizeYomi` で正規化してから比較

**UI 変更**：
- プレースホルダーを「氏名で検索（漢字・ふりがな）」に変更
- 既存の漢字検索動作は維持（破壊的変更なし）

### 3.3 F6: 50音タブ（パターン α + あ）

**配置**（パターン α：検索ボックスの下に50音タブ）：

```
過去参加者から選ぶ                         登録 87名 [▲ 閉じる]
─────────────────────────────────────────
[🔍 検索: ___________________________ ]
[全][あ][か][さ][た][な][は][ま][や][ら][わ][他]
─────────────────────────────────────────
☐ 佐藤太郎    Aクラス    2026-04
☐ 鈴木一郎    Bクラス    2026-04
...
```

**タブ定義**：
- 全 / あ / か / さ / た / な / は / ま / や / ら / わ / 他（計12タブ）
- 「全」がデフォルト（既存動作と同じ）
- 各行タブは yomi の先頭文字に基づく
- **「他」タブには yomi 空の member を含める**（パターン あ）
  - 加えて、yomi はあるが「あ〜わ行のどれにも該当しない」人（外国名等）も含める

**タブ + 検索の併用**：
- タブ選択中も検索ボックスでの絞り込みが効く
- 例：「さ行」タブ + 検索「いとう」→ さ行 ∩ 「いとう」一致 = 0件などの動作になる
- AND 条件で絞り込む（OR ではない）

**タブのソート/表示**：
- 全タブ：既存通り（最終参加日降順、同日なら氏名昇順）
- 各行タブ：yomi 昇順（同 yomi なら氏名昇順）
- 「他」タブ：yomi 空が先頭、後方に外国名等

### 3.4 normalizeYomi (新規ユーティリティ)

ふりがなの正規化関数：

```javascript
function normalizeYomi(yomi) {
  if (typeof yomi !== 'string') return '';
  // 1. 前後空白除去
  // 2. カタカナ → ひらがなに変換（ヤマダ → やまだ）
  // 3. 全角空白を除去（途中空白も削除、ふりがなには空白不要）
  // 4. 連続空白を単一に → 結果的に削除
  // 5. 長音符（ー）はそのまま保持
  // 6. 大文字（ぁ→ぁ）等の小書き文字はそのまま保持
  return yomi
    .replace(/^\s+|\s+$/g, '')
    .replace(/[ァ-ヶ]/g, (m) => String.fromCharCode(m.charCodeAt(0) - 0x60))
    .replace(/[\s　]/g, '');
}
```

**設計意図**：
- ユーザーがカタカナで入力しても、ひらがなで入力しても、内部的に統一
- 50音タブの先頭文字判定が安定する
- 検索時の比較も同じ関数で正規化

### 3.5 50音先頭文字判定 (新規ユーティリティ)

```javascript
function getYomiInitialRow(yomi) {
  // normalizeYomi 済みの yomi から行を判定
  if (!yomi) return 'other';
  var first = yomi.charAt(0);
  // 各行のひらがな先頭判定
  if ('あいうえおぁぃぅぇぉ'.indexOf(first) >= 0) return 'a';
  if ('かきくけこがぎぐげご'.indexOf(first) >= 0) return 'ka';
  if ('さしすせそざじずぜぞ'.indexOf(first) >= 0) return 'sa';
  if ('たちつてとだぢづでどっ'.indexOf(first) >= 0) return 'ta';
  if ('なにぬねの'.indexOf(first) >= 0) return 'na';
  if ('はひふへほばびぶべぼぱぴぷぺぽ'.indexOf(first) >= 0) return 'ha';
  if ('まみむめも'.indexOf(first) >= 0) return 'ma';
  if ('やゆよゃゅょ'.indexOf(first) >= 0) return 'ya';
  if ('らりるれろ'.indexOf(first) >= 0) return 'ra';
  if ('わをん'.indexOf(first) >= 0) return 'wa';
  return 'other';  // 数字・アルファベット・記号等
}
```

### 3.6 Codex Minor #1: 破損マスタ状態でのマイグレ時の確認表示

**現状（A-1 v5）**：
- マスタが破損している状態でも、マイグレーションウィザードは普通に起動・実行できる
- マイグレーション後は破損マスタが新しいマスタで上書きされ、復旧する
- これは仕様通りの動作だが、**ユーザーが破損に気付かないまま操作する**点が懸念

**A-2 改善**：
- マイグレーションウィザード起動時、破損マスタを検出していたら **冒頭で警告表示**

```
┌─ 過去大会の統合 ──────────────────────────┐
│ ⚠️ 既存の支部マスタが破損していました。     │
│ このマイグレーションで再構築されます。      │
│ （既存マスタは破棄されます）                │
│                                            │
│ 過去の大会JSONファイルを読み込んで...       │
└────────────────────────────────────────┘
```

実装：
- `loadBranchMaster()` の `_loaded_with_corruption` フラグを `openMigrationWizard()` で参照
- フラグ立っていれば警告セクションをモーダル冒頭に追加
- フラグなしなら従来通り（警告なし）

### 3.7 Codex Minor #2: crypto 非対応ブラウザでのユーザー通知強化

**現状（A-1 v5）**：
- `generateMemberId()` で `crypto.randomUUID` 不在時に throw
- 該当参加者は同期スキップされ、ユーザーには通知が薄い

**A-2 改善**：
- `generateMemberId()` の throw を `syncBranchMasterOnSave()` または呼び出し側でキャッチ
- 「お使いのブラウザは古いため、参加者の自動登録ができませんでした。ブラウザ更新を推奨します」を showMsg で通知
- 大会JSON保存自体は継続（既存仕様維持）

実装：
- `updateBranchMasterFromTournament` 内の try-catch で `generateMemberId` 不在エラーを判定
- 専用エラーメッセージかをチェックし、専用通知を出す
- 一度通知したら同セッション内で再通知しない（フラグ管理）

## 5. テスト計画

### 5.1 既存テスト維持（最重要）

A-1 v5 時点の全テスト（既存50 + 支部マスタ機能111 = 計161）を **すべて緑のまま維持**。

### 5.2 新規追加テスト

#### 5.2.1 normalizeYomi
- カタカナ → ひらがな変換（「ヤマダ」→「やまだ」）
- 前後空白除去
- 全角・半角空白の除去
- 長音符（ー）保持
- 小書き文字（ぁ ぃ ぅ）保持
- 空文字・null・undefined・非文字列

#### 5.2.2 getYomiInitialRow
- 各行（あ〜わ）の判定
- 濁音・半濁音（が、ぱ）の正しい行判定
- 小書き文字（ぁ、ょ）の正しい行判定
- 「他」判定（yomi 空、外国名、数字、記号）

#### 5.2.3 ふりがな検索（F5b）
- 漢字一致のみ → ヒット
- ふりがな一致のみ → ヒット
- 両方一致 → 1回だけヒット（重複なし）
- どちらも不一致 → ヒットせず
- カタカナ入力での検索（「サトウ」→「さとう」と同じヒット）

#### 5.2.4 50音タブ（F6）
- 「全」タブで全 member 表示
- 「あ」タブで yomi 先頭が あ行 のみ表示
- 「他」タブで yomi 空 + あ〜わ行外の member 表示
- タブ + 検索の AND 条件
- yomi 空の member が「他」タブで見える

#### 5.2.5 ふりがな入力ダイアログ（F4）
- yomi 空が0名 → ダイアログ非表示で即追加
- yomi 空が複数名 → ダイアログ表示
- 入力された yomi がマスタに保存される
- 空欄のまま「追加する」 → yomi 空のまま追加
- 「キャンセル」 → 何も追加されない
- 既に yomi がある member はダイアログに含まれない

#### 5.2.6 Codex Minor 反映テスト
- 破損マスタ状態でマイグレウィザード起動 → 警告表示
- crypto 不在環境で参加者追加時の通知

### 5.3 性質テスト

- 任意の N 名を yomi 付きで登録 → 50音タブ各行の合計が N と一致
- 同じ yomi の異なる氏名（同読みの別人）が同じタブに入る

### 5.4 既存大会運営の非破壊（A-1 と同じ哲学）

- yomi なしの A-1 マスタを A-2 で開いても全機能動く
- A-2 で yomi を追加したマスタを A-1 で開いても壊れない（schema_version 同じ）
- 既存の漢字検索は引き続き動作する

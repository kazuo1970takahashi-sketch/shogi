## 0. 背景

A-3完了後の実機運用テストで以下が判明した。

1. **ふりがな空振り問題**：登録画面に氏名（漢字）入力欄しかなく、ふりがなはマスタタブからしか入力できない。結果、運用でふりがなが入らず50音タブが機能しない。
2. **member/grade不可視問題**：マスタにmember/grade（支部員区分・中学生以下区分）を持たせているが、マスタ編集画面で表示・編集できない。
3. **削除済み member 復元手段なし**：A-3で「A-4以降」と先送りした項目。
4. **ふりがな未入力者の可視化なし**：自動取得を導入しても全員に入る保証はないため、未入力者を可視化する仕組みが必要。

A-4 はこれら実運用上の穴を塞ぎ、月例大会運営でアプリが「使い物になる」状態を完成させる。

## 3. 機能仕様

### 3.1 登録画面ふりがな入力欄＋IME自動取得（F9）

#### 3.1.1 UI

```
氏名: [山田太郎          ]   クラス: [A▼]   [追加]
ふりがな: [やまだたろう    ]
```

- 氏名欄の下にふりがな入力欄を配置
- 入力欄は font-size: 16px（A-2.6 iPhone Safari ガイドライン準拠）
- placeholder: `ふりがな（自動入力されます）`
- 既存のサジェスト機能とは独立（サジェスト選択時はマスタの yomi が反映される）

#### 3.1.2 IME自動取得ロジック

CompositionEvent を使ってひらがな入力履歴から最終的なふりがなを取得する。

**動作原理**：
- ユーザーが氏名欄に「やまだたろう」と打って「山田太郎」に変換
- compositionupdate イベントで「やまだたろう」を保存
- compositionend で確定後、保存していた値をふりがな欄に反映

**実装方針**：
1. `compositionstart`：ひらがなバッファをリセット、現在のふりがな欄値も記憶（手動編集との衝突回避）
2. `compositionupdate`：`event.data` がひらがな（U+3040〜U+309F）のみで構成されていればバッファに記録（最後の値を保持）
3. `compositionend`：直前のバッファを yomi 欄に追記（既存値の末尾に）。yomi欄の値はnormalizeYomi（既存関数）で正規化。

**仕様詳細**：
- 苗字と名前を別々に変換した場合 → バッファに2回追加されて累積される
- yomi 欄を手動で変更した場合 → 自動取得は上書きしない
- IMEを使わず直接漢字をペーストした場合 → 自動取得不可、手動入力に依存
- カタカナ入力（ヤマダ）の場合 → normalizeYomi がひらがなに変換

#### 3.1.3 マスタ更新時の扱い

新規参加者の氏名＋ふりがな両方が入力されている場合：
- `createMemberFromParticipant` で yomi をマスタに記録（既存ロジック、変更なし）

サジェストから選んだ参加者の場合：
- マスタの yomi が氏名欄選択時に自動入力される（既存）
- ユーザーが手動でふりがなを変更した場合、updateBranchMasterFromTournament で yomi を更新する

**禁止事項**：参加者の player オブジェクトには yomi フィールドを追加しない。マスタ側のみ。

#### 3.3.2 ロジック変更

`applyMasterMemberEdit(memberId, newName, newYomi, master)` を拡張：
- `applyMasterMemberEdit(memberId, newName, newYomi, newMember, newGrade, master)` に変更

**後方互換**：
- 第4引数が文字列でなくオブジェクトの場合（newMember/newGradeを受け取る）→ 新シグネチャ
- 既存テスト（A3-S2-edit-*）は新シグネチャに合わせて修正
- または、5番目以降の引数を追加して既存テスト維持

→ **採用案**：オプション引数を追加して既存テストを破壊しない方針
```javascript
applyMasterMemberEdit(memberId, newName, newYomi, master, options) 
// options = {member: 'member'|'other', grade: 'ippan'|'chu'} （省略可）
```

`options` 省略時は member/grade を変更しない（既存テスト互換）。
`options` 指定時のみ更新。

### 3.6 スマホレイアウト揺れ修正（F10）

#### 3.6.1 現象

iPhone 16 Plus（論理幅 430px）でタブを切り替えると、各タブの左右マージンが微妙に異なって見える。コンテンツの幅がタブごとに違い、画面全体が揺れて見える。

#### 3.6.2 想定原因

複数の原因が複合している可能性：

1. **コンテンツ幅の不揃い**：タブごとにテーブル・ボタン・モーダル等の最大幅が異なる
2. **横スクロール発生**：一部タブだけ overflow-x: scroll になっている
3. **box-sizing の不整合**：padding と width の計算ミス
4. **viewport meta の設定漏れ**：initial-scale や user-scalable
5. **Safari セーフエリア**：iPhoneのノッチ・ホームバー領域の扱い差

#### 3.6.3 調査・修正方針

**Stage 6 冒頭で実機 DevTools 検証**：
1. iPhone 16 Plus（または近似サイズ：375px / 414px / 430px）で各タブを開く
2. Safari Web Inspector でタブごとの `<body>` および `.tab-content` の幅を測定
3. 横スクロールバーの有無を確認（`document.documentElement.scrollWidth > window.innerWidth`）
4. 揺れの原因となっている要素を特定

**修正パターン例**：
- ルート要素に `max-width: 100vw; overflow-x: hidden` を追加
- 各タブのコンテナに統一クラスを適用（max-width / margin auto / padding 統一）
- テーブル系には `table-layout: fixed` または `min-width` を再検討
- viewport meta に `viewport-fit=cover` 追加（セーフエリア対応）

**禁止事項**：
- 既存のレスポンシブ対応（A-2.6 で確定したiPhone Safari UXガイドライン）を破壊しない
- font-size 16px ルールは維持
- 「揺れる」という曖昧な原因を1つに決め打ちせず、実機で再現してから修正

#### 3.6.4 e2eテスト

Playwright の `--project=mobile-375` で：
- 各タブ（参加者登録 / 対局 / 最終結果 / マスタ）を開く
- `document.documentElement.scrollWidth === window.innerWidth` を確認（横スクロールなし）
- 各タブの `<body>` 幅が同一であることを確認

## 5. テスト戦略

### 5.1 単体テスト追加項目

**Stage 1-2（ふりがな自動取得）**：
- 単体テストは限定的（CompositionEventはブラウザ依存のためe2e中心）
- normalizeYomi の既存テストでカバー

**Stage 3（マスタ編集）**：
- `applyMasterMemberEdit` に options 引数を追加した場合の動作
- options 省略時は既存挙動維持
- options.member 指定で member 更新
- options.grade 指定で grade 更新
- 不正値（'foo' 等）はエラー

**Stage 4（復元）**：
- `applyMasterMemberRestore` 通常成功
- 不存在id → not_found
- deleted=false → not_deleted
- 復元後の deleted=false / deleted_at=null

**Stage 5（フィルタ）**：
- `applyQuickFilter` に QUICK_FILTER_NO_YOMI 追加
- yomi 空 = 未入力扱い
- yomi 空白文字列も未入力扱い（normalizeYomi 適用後）

### 5.2 e2eテスト追加項目

**Stage 1-2**：
- ふりがな入力欄が表示される
- 手動入力した値が保存される
- IME入力時に自動でふりがなが入る（Playwright keyboard.imeで再現）

**Stage 3**：
- マスタ一覧に区分が表示される
- 編集モーダルで支部員区分・中学生区分を変更できる
- 変更が localStorage に保存される

**Stage 4**：
- 削除済み表示トグルで一覧に削除済みmemberが出る
- 復元ボタンで deleted=false になる
- 過去参加者パネル・サジェストで再び表示される

**Stage 5**：
- マスタタブのサマリーに「ふりがな未入力: N名」表示
- ⚠️未入力バッジが出る
- ふりがな未入力フィルタが動作する

**Stage 6（レイアウト揺れ）**：
- mobile-375プロジェクトで各タブを開いて横スクロールが発生しないこと
- 各タブの body 幅が一致すること

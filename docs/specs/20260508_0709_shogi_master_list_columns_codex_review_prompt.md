## 1. 背景・目的

2026-05-08 実機確認で、マスタ一覧画面が 7 列(氏名/ふりがな/区分/前回クラス/最終参加/回数/操作)構成のため iPhone 375px で氏名・区分・「未入力」警告ラベル等がすべて縦書き化していることが判明(髙橋さん指摘)。

過去参加者パネル(A-4 系で対策済)とは別画面のため、L-3/L-4 ヘルパー導入後も未対策のまま残っていた。本 PR で **5 列構成(氏名 / 支部員区分 / 中学生以下区分 / 編集 / 削除)** に変更し iPhone 375px 縦書き化を解消。F7 編集モーダルは変更せず、削除した列(ふりがな / 前回クラス / 最終参加 / 回数)は F7 内で引き続き確認・編集可能。

## 2. 変更内容

### 2.1 production 修正 (`shogi_v4.html`、+21 / -25 行、コミット `707aa17`)

- thead を 7 列 → 5 列に変更(氏名 / 支部員区分 / 中学生以下区分 / 編集 / 削除)
- 「区分」セルを「支部員区分」(member: 支部員 / 他)+「中学生以下区分」(grade: 中学 / 一般)の 2 セルに分離
- 中学生以下区分は **「中学」/「一般」両方を常時表示**(従来は中学のみ表示で「一般」は空欄)
- 操作列を「編集」/「削除」の 2 セルに分離(削除済み行は「復元」+「削除日」の 2 セル)
- CSS クラス: `master-cell-member` 新設、`master-cell-grade` は流用、`master-cell-edit` / `master-cell-delete` 新設
- `master-cell-last-class`(前回クラス)、`master-no-yomi-badge`(ふりがな未入力バッジ)は撤廃

### 2.2 設計判断

**(a) 操作列を 2 セル分離(仕様書 §2.1 解釈)**

仕様書 §2.1 表で「編集 / 削除」が別行で記述されており、編集列・削除列を別々の `<td>` にする解釈を採用。これにより `master-cell-edit` / `master-cell-delete` での個別 e2e ターゲティングが可能。削除済み行は「編集セル → 復元ボタン」「削除セル → 削除日」の対称構造で 5 列を維持。

**(b) 中学生以下区分を「中学」/「一般」常時表示に変更**

従来は `gradeLabel = (grade === 'chu') ? '中学' : ''` で「一般」は空欄だったが、列を分離した結果空欄セルは UX 上違和感が出るため `'中学'` / `'一般'` 両方を表示。`grade` フィールド(必須)が常に「中学」or「一般」の二択であることをセル単独で示す。

**(c) ふりがな未入力バッジ(`master-no-yomi-badge`)は撤廃、サマリーは維持**

ふりがな列を撤廃したため、行内の `master-no-yomi-badge` は存在しない。一方、マスタ画面上部の「登録: N名(うちふりがな未入力: M名)」サマリー(`#master-no-yomi-summary`)は production code で残しているため引き続き表示。ふりがな未入力の識別は **F7 編集モーダル内 + サマリー** で担保。

### 5.2 Devil's Advocate(仕様書 §6)必須回答

1. **同姓同名識別**:
   - 沼津支部の現状マスタで同姓同名の実例は不明だが、月例大会の規模(数十名)で発生する可能性は低い
   - 万一発生した場合、F7 編集モーダル内で識別可(name + yomi + member + grade + 履歴情報を確認可能)
   - 一覧で識別したい要件が出れば次 Sprint で「氏名セル内に小さくふりがな表示」案(仕様書 §7)を実装

2. **検索フィルタ動作**:
   - 事前調査の結果、**マスタ画面に検索フィルタは存在しない**(`grep search master` で hit なし)
   - 仕様書 §2.4 の「検索フィルタ動作維持」は適用対象なし(影響ゼロ)

3. **削除列の影響範囲**:
   - 「前回クラス」: 過去参加者パネル(運営前に確認可)+ F7 編集モーダルで識別可
   - 「最終参加」「回数」: F7 編集モーダル内の履歴情報セクションで確認可
   - 運営クリティカルパス(参加者登録 → 対局 → 結果)はマスタ一覧を経由しないため影響なし

4. **iPhone 375px 縦書き化解消の確証**:
   - 5 列構成: padding(8px × 2 × 5 = 80px)+ border(僅か)+ 氏名セル + 区分 2 セル + 編集 / 削除ボタン
   - L-4 ヘルパーで通常氏名セル < 60px / 長氏名セル < 100px / 支部員区分セル < 60px を実測 assert
   - L-3 ヘルパーで page 全体に水平 overflow なしを assert(通常 + 長氏名)

5. **mobile-375 visual snapshot 反映確認**:
   - 新規 `mobile-master-list-5cols-375-mobile-375-darwin.png`(94821 bytes、コミット `944de56` 含む)
   - 既存 `mobile-master-edit-modal-375-mobile-375-darwin.png` は更新(112853 → 75439 bytes、F7 編集モーダル前の画面遷移差異反映)

6. **A-4.4 の轍を踏んでいないか**:
   - 列削除で行内要素が **減る** ので flex 圧迫リスクは下がる方向
   - 新設「中学生以下区分」セルは 1 列(2-3 文字程度)で行幅にほぼ影響しない
   - L-3 / L-4 ヘルパーで構造的検出網も確立済(mobile-375 project の visual regression 含む)

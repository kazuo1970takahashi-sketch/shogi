## §1 背景・目的

2026-05-08 実機確認(commit `6b72f2a`)で、マスタ一覧画面が 8 列構成のため iPhone 375px で氏名・ふりがな・区分・「未入力」警告ラベルがすべて縦書き化していることが判明。スクリーンショットで確認:

- 「SCSK山田中」「かさら「たあ」のような氏名が縦に積まれて読みにくい

- ふりがな列の「未入力」警告ラベルも縦書き(未/入/力)

**致命的ではないが見栄えが悪く、髙橋さんから対応希望**。

過去参加者パネル(A-4 系で対策済)とは別画面のため、L-3/L-4 ヘルパー導入後も未対策のまま残っていた。

## §2 機能仕様

### 2.1 マスタ一覧の列構成変更

| 変更前(8 列) | 変更後(5 列) |
|---|---|
| 氏名 | 氏名 |
| ふりがな | (削除、編集モーダルで確認) |
| 区分 | **支部員区分**(リネーム) |
| (なし) | **中学生以下区分**(新設) |
| 前回クラス | (削除、編集モーダルで確認) |
| 最終参加 | (削除、編集モーダルで確認) |
| 回数 | (削除、編集モーダルで確認) |
| 編集 | 編集 |
| 削除 | 削除 |

### 2.2 列の表示内容

| 列 | フィールド | 表示値 |
|---|---|---|
| 氏名 | `name` | そのまま表示 |
| 支部員区分 | `member` | "支部員" / "他" |
| 中学生以下区分 | `grade` | "中学" / "一般" |
| 編集 | (button) | 「編集」 |
| 削除 | (button) | 「削除」 |

### 2.3 編集モーダル(F7)変更なし

F7 編集モーダルでは現状通り **すべての項目**(氏名 / ふりがな / member / grade / last_class / note 等)を編集可能。マスタ一覧から削除した「ふりがな」「前回クラス」「最終参加」「回数」は **F7 編集モーダル内で確認・編集可能**。

### 2.4 検索フィルタ(マスタ画面に存在する場合)

- 検索フィルタの動作は **変更しない**(ふりがな検索が機能している場合は維持)
- ふりがなが一覧で見えなくても、検索でヒットした行は氏名で識別可能

## §5 リスク・移行戦略

### 5.1 同姓同名識別

ふりがなが一覧で見えなくなることで、同姓同名(例:「山田太郎」が複数)の識別が一覧でできなくなる。**編集モーダル内で識別** する運用となる。沼津支部の規模(数十名)であれば実用上問題なし。

### 5.2 検索フィルタ

マスタ画面に検索フィルタが存在する場合、現状維持(ふりがな検索が機能していれば残す)。検索結果は氏名 + 支部員区分 + 中学生以下区分で識別可能。

### 5.3 既存 e2e の破壊リスク

- a4_3 spec の「前回クラス」列 assert を削除する箇所がある
- 1〜2 件の修正で済む想定、`grep` で該当箇所を特定

## §7 想定外時の対応

- ふりがなが一覧で必要との判断が出た場合: 「氏名」セル内に小さく表示する形(2 段表示)で対応
- 検索フィルタが想定より複雑な場合: 別 PR で検索 UX 改善を分離
- マスタ一覧画面の構造が想定より複雑な場合: 段階分割

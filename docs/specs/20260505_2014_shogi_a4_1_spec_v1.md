## 4. 想定される揺れの原因仮説

※実機計測前の仮説。Stage 1 で確定または棄却する。

| # | 仮説 | 影響範囲 | 対策方針 |
|---|------|----------|----------|
| 1 | `100vh` と `100dvh` の差異（iOS Safari アドレスバー表示/非表示で vh が変動） | 全画面・特に縦スクロール領域 | `100dvh` 採用 + `100vh` フォールバック |
| 2 | `safe-area-inset` 未適用 | 下部固定要素（フッター/ボタン） | `env(safe-area-inset-bottom)` 適用 |
| 3 | IME 表示による visualViewport 変動 | 入力欄付近・特に登録画面 | `visualViewport` API or scroll padding |
| 4 | サジェスト/モーダル表示時の DOM 挿入による reflow | 登録画面（A-3 サジェスト）/ マスタ編集モーダル | `position: absolute` or `transform` 化 |
| 5 | フォント読込による FOUC/CLS | 全画面 | `font-display: optional` or 事前読込 |
| 6 | A-4.2 の A/B クラスボタン挿入によるレイアウト影響 | 過去参加者パネル・サジェスト | min-height 固定 or grid 化 |
| 7 | ふりがな入力欄（A-2）の表示/非表示切り替え | 登録画面 | 高さ予約 or 常時表示化 |

## 1. A-3 スコープ（確定）

### 1-1. 登録画面×マスタ接続（新規追加・最優先）

**目的**: 受付時のボトルネック（幹事による手入力）を常連参加者に限り大幅削減する

**確定仕様**:

| 項目 | 内容 | 決定理由 |
|------|------|---------|
| 候補表示方式 | 氏名入力欄にインラインサジェスト | 20名規模では苗字2〜3文字で絞れる。一覧より速い |
| 引き継ぎ項目 | 氏名・クラス・支部員区分・中学生以下区分（全フィールド） | 受付負荷の最大削減のため全引き継ぎ |
| 引き継ぎ後の修正 | 追加前に全フィールド変更可能 | 区分変更に対応 |
| クラス変更の対応 | 削除→再登録 | 1大会1〜2名の稀なケース。複雑な仕組みは作らない |

**対象**: 常連（過去参加者）。初参加者はカード見ながら手入力（従来通り）。

**見送り**:
- B案（50音タブ一覧から選ぶ）→ 20名規模には過剰
- QRコード受付 → 参加者層への依存度が高い
- タッチパネル受付 → 導入コストが高く同じ課題

### 1-2. F7 マスタ管理画面（編集・削除）

**F7-a 編集**:
- 編集対象: `name + yomi` のみ
- 参加履歴（attendance_count / last_attended 等）は編集不可
- 同名重複は warning のみ（自動統合・ブロックはしない）

**F7-b 削除**:
- 方式: tombstone（`deleted: true` + `deleted_at`）
- 物理削除はしない
- 削除済みは過去参加者パネル・登録画面サジェストから非表示
- 復元 UI は A-4 以降（A-3 では作らない）

**誤削除時の救済路（v7.2 修正）**:
- 正式救済路: 事前エクスポートしたバックアップを上書きインポートで戻す
- LocalStorage 直接編集は開発者向けの最終手段であり、通常運用には含めない

### 1-3. F8 マスタバックアップ（エクスポート/インポート）

**v7.2 最重要修正: データ種別の分離**

| 操作 | 対象データ | 用途 |
|------|-----------|------|
| F8 エクスポート/インポート | branch master 形式（schema_version + members） | 端末移行・バックアップ・復元 |
| 既存マイグレ機能 | 大会データ形式（players を含む tournament） | 過去大会から支部マスタを構築 |

**F8 は branch master 形式のみを扱う。大会データの統合は既存マイグレ機能の責務であり、F8 では行わない。**

#### F8-a エクスポート
- 対象: `shogi_branch_master`（schema_version / updated_at / members）
- tombstone を含めてエクスポートする
- エクスポート直後に「端末移行や誤削除に備えて保管してください」を案内

#### F8-b 上書きインポート
- 入力形式: branch master 形式のみ（members を持つ JSON）
- branch master 形式以外はエラー表示して中断
- 現在のマスタを完全に置き換える
- **確認ダイアログ（v7.2 強化）**:
  ```
  現在のマスタをすべて置き換えます。
  この操作は元に戻せません。
  事前にマスタをエクスポートしていない場合はキャンセルしてください。
  [現在のマスタを破棄して上書き] [キャンセル]
  ```
- バックアップ時点の tombstone 状態も含めて完全復元される

#### F8-c マージインポート
- 入力形式: branch master 形式のみ
- `member.id` を基準に既存 member と統合する
- `name + yomi` の一致だけで自動統合しない
- **tombstone の挙動（v7.2 確定）**:
  - マージインポートで `deleted=true` の member を `deleted=false` に自動復元しない
  - 安全側に倒し、`deleted=true` を維持する
  - 復元は A-4 の復元 UI で扱う
- `id` が異なる member は別 member として追加

#### 削除された性質テスト（v7.2）
- ~~F8 エクスポート → 削除 → インポート（マージ）で削除分が復活する~~ → 削除

#### 追加する性質テスト（v7.2）
- F8 エクスポート → 削除 → インポート（上書き）でバックアップ時点の状態に戻る
- F8 マージインポートで同一 id の tombstone が勝手に復活しない
- branch master 形式以外を F8 インポートしようとした場合にエラー

### 1-4. クイックフィルタ（v7.2 名称修正）

| フィルタ名 | 内部定義 |
|-----------|---------|
| 前回参加 | member.last_attended が最終大会日と一致 |
| 3ヶ月以内 | member.last_attended が直近3ヶ月以内 |

## 4. 実装禁止事項

1. schema_version を変更しない
2. F8 で大会データ（players を含む tournament 形式）を処理しない
3. `updateBranchMasterFromTournament` を F8 のロジックとして使わない
4. マージインポートで tombstone を自動復元しない
5. name + yomi の一致だけで member を自動統合しない
6. 既存のマイグレ機能（過去大会→マスタ統合）の責務を変えない
7. 参加履歴（attendance_count / last_attended 等）を F7 編集画面で編集可能にしない
8. 復元 UI を A-3 に実装しない
9. A-4 送り項目を先取り実装しない
10. build/bind 分離パターンを壊さない

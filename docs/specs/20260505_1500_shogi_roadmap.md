## A-3 スコープ（確定）

### ★ 登録画面×マスタ接続（最優先・受付軽減の核心）

**狙い**：受付時のボトルネックは幹事による参加者情報の手入力。月例大会は常連が多いため、過去参加者をマスタから即呼び出せれば入力負荷を大幅に削減できる。

**確定仕様（2026-05-05）**：

| 項目 | 内容 |
|------|------|
| 候補表示方式 | A案：氏名入力欄にサジェスト表示（インライン） |
| 引き継ぎ項目 | 氏名・クラス・支部員区分・中学生以下区分（全フィールド） |
| 引き継ぎ後の修正 | 追加前に全フィールド変更可能 |
| クラス変更の対応 | 削除→再登録（1大会1〜2名の稀なケースのため複雑な仕組みは作らない） |

**確定の根拠**：
- 参加者は最大20名規模 → サジェストで苗字2〜3文字で絞れる、一覧不要
- クラス変更はほぼなし（成長期の子供で年1〜2名程度）
- 削除ボタンは既存UIに存在するため再登録フローは実現可能

**見送り**：
- B案（50音タブ一覧から選ぶ）→ 20名規模には過剰、タップ回数が増える

## A-4 スコープ（v1仕様書ベース）

### 実機運用テストで判明した5項目

| # | 項目 | 概要 |
|---|------|------|
| 1 | 登録画面ふりがな入力欄＋IME自動取得 | CompositionEventでふりがな自動入力、手動編集可 |
| 2 | マスタ一覧に区分列追加 | member/grade を一覧表示（読み取り専用） |
| 3 | マスタ編集モーダルに区分追加 | name + yomi に加えて member/grade も編集可能 |
| 4 | 削除済みmember復元UI | tombstoneを表示・復元できる |
| 5 | ふりがな未入力者可視化 | サマリー・バッジ・フィルタで未入力を可視化 |

**スコープ外**：ふりがな自動「推定」（kuromoji.js）、ローマ字検索、異体字統合、半角カナ対応

**Stage構成**：6ステージ（UIのみ→IME自動取得→マスタ拡張→復元UI→未入力可視化→全体テスト）

## バックログ（将来オプション・現時点では着手しない）

### QRコード受付
参加者が自分のスマホでQRを読んで入力する方式。幹事の入力ゼロになるが、参加者側に操作を求める。月例大会の参加者層（年配の方も多い）を考えると全員対応は現実的でない。

**見送り理由**：参加者層への依存度が高く即効性が低い。マスタ活用の完成を優先する。

### タッチパネル受付
**見送り理由**：QRより導入コストが高く、同じ課題を持つ。

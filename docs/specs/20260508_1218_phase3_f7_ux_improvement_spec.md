# Phase 3 Mini 仕様書: F7 編集モーダル UX 改善

- 対象: shogi_v4.html(沼津支部運営ツール)
- 関連: Phase 1 / Phase 2 マージ済(main commit `c72c690`)、Postmortem v2(L-1〜L-4)
- 目的: F7 編集モーダルの操作性向上(初期表示で保存ボタンが見えない問題の解消 + 文言の不整合解消)
- 区分: Mini Spec / Gate Quick Review(P0 ではない、既存挙動破壊観点のみ)

---

## 1. 目的・背景

Phase 1 で city フィールドを追加した結果、F7 編集モーダルの縦長が増し、**初期表示状態で保存/キャンセルボタンが viewport 内に見えない問題が顕在化**(髙橋さん 5/8 実機確認時にフィードバック)。加えて、モーダル冒頭の説明文「氏名ふりがな区分が編集できます。参加履歴は変更されません。」が、後続フェーズ(前回クラス、city)を反映していない**不整合状態**。

両方を Mini Sprint で解消する。本フェーズは 5/10 本番直前の UX 改善で、機能要件には影響しない。

## 2. スコープ

### 2.1 改善 1: 履歴情報を折りたたみ(▼開く)

F7 編集モーダル内の「履歴情報」セクションをアコーディオン式に変更。

- デフォルト: **閉じた状態**(初期表示で保存/キャンセルボタンが viewport 内に収まる)
- 「▼ 開く」タップで展開、再タップで閉じる
- UI パターンは既存「過去参加者から選ぶ」セクションと同一(同じヘルパー関数 / CSS パターンを再利用、Plan Mode で実装手段確認)

### 2.2 改善 2: 説明文削除

F7 編集モーダル冒頭の説明文「氏名ふりがな区分が編集できます。参加履歴は変更されません。」を撤去。

理由:
- 各入力欄の label と placeholder で意図は伝わる
- 後続フェーズ(前回クラス追加、city 追加)を反映していない不整合
- モーダルの簡潔化

## 3. UI 変更詳細

| 場所 | 変更前 | 変更後 |
|---|---|---|
| F7 モーダル冒頭 | 説明文「氏名ふりがな…」表示 | 説明文撤去 |
| F7 モーダル下部「履歴情報」 | 直接表示 | 「▼ 履歴情報を開く」トグル + 折りたたみコンテンツ |

説明文以外の入力欄(氏名 / ふりがな / 支部員区分 / 中学生以下 / 前回クラス / city)は **変更なし**。

## 4. 受け入れ条件 (Quick-level e2e)

1. F7 モーダル初期表示で **履歴情報セクションが折りたたまれている**(アコーディオン閉じ状態)
2. 「▼ 開く」タップで履歴情報展開、再タップで閉じる
3. F7 モーダル初期表示で **保存/キャンセルボタンが viewport 375px 内に表示** されている(本フェーズの主目的)
4. 説明文「氏名ふりがな区分が…」が表示されない
5. F7 全フィールド(氏名 / ふりがな / 支部員区分 / 中学生以下 / 前回クラス / city)の保存・読み込みが Phase 1 / Phase 2 と同じ動作(**既存挙動破壊なし**)
6. 既存 e2e 緑維持(Phase 2 マージ後の 682 + Phase 3 追加分)

## 5. Out of Scope

- 保存/キャンセルボタンの sticky 固定(別案、本フェーズでは採用しない)
- 履歴情報自体の削除(完全削除は将来検討、本フェーズでは折りたたみのみ)
- 履歴情報の表示内容変更(直近 N 件絞り込み等)
- F7 以外のモーダル(新規追加・削除確認・マスタリセット・Phase 2 import 等)の改修

## 6. リスク(P1 以下)

- 折りたたみ実装で既存「履歴情報」表示ロジックを壊す → e2e #5 で防止
- 既存「過去参加者から選ぶ」のアコーディオンと見た目が整合しない → 同一ヘルパー / CSS パターン使用
- mobile-375 visual snapshot の F7 系で意図しない差分 → snapshot 更新範囲を PR Body に明記
